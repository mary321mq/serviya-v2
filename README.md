# SERVIYA

Proyecto final de Sistemas Distribuidos.

ServiYa es una plataforma distribuida para gestion de servicios tecnicos, solicitudes, cotizaciones, pagos, comprobantes, tecnicos, resenas, notificaciones y observabilidad.

## Documentacion tecnica

La documentacion esta preparada con MkDocs.

```bash
pip install -r requirements-docs.txt
mkdocs serve
```

Abrir:

```text
http://localhost:8000
```

## Configuracion obligatoria de Topics en GitHub

Antes de la entrega final, el equipo debe configurar los topics del repositorio siguiendo el estandar oficial solicitado para repositorios academicos.

Formato requerido:

```text
campus-[sede], semestre-2026-1, linea-software, tipo-pi, dist, seccion-[seccion], grupo-[numero]-[nombre]
```

Ejemplo para ServiYa:

```text
campus-juliaca, semestre-2026-1, linea-software, tipo-pi, dist, seccion-a, grupo-01-serviya
```

Ruta en GitHub:

```text
Repositorio -> About -> Settings icon -> Topics
```

Verificacion:

- El repositorio debe mostrar todos los topics visibles en la seccion About.
- Los topics deben estar en minusculas.
- No deben usarse espacios.
- Usar guiones `-` para separar palabras.
